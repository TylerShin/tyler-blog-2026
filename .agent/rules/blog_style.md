# 블로그 포스트 작성 가이드

이 저장소에 블로그 글을 작성할 때는 다음 규칙을 따르세요.

## 1. 어조 및 스타일
- **건조하고 간결하게**: 너무 딱딱하지는 않되, 감정을 절제한 톤을 유지합니다.
- **종결 어미**: 문장을 명사형으로 끝맺습니다 ("~함", "~임", "~됨", "~있음", "~가능함" 등).
  - 좋음: "설치할 수 있음.", "편리함.", "해결됨."
  - 나쁨: "설치하시면 됩니다.", "편리합니다." (너무 격식 차림), "설치해." (너무 반말)

## 2. 글 구조
1.  **Frontmatter**:
    - `title`: 명확하고 설명적인 제목.
    - `summary`: 한 줄 요약.
    - `date`: YYYY-MM-DD 형식.
    - `tags`: 구체적인 태그 사용 (3번 항목 참조).
2.  **TL;DR**:
    - Frontmatter 직후에 반드시 `## TL;DR` 섹션 배치.
    - 핵심 명령어/코드를 바로 실행할 수 있도록 제공 (예: 설치 명령어).
3.  **본문 (## 본문)**:
    - 상세 설명, 배경, 사용법 등을 기술.
    - 하위 섹션은 H3(`###`), H4(`####`) 사용.
4.  **References**:
    - 글 하단에 `## References` 섹션 배치.
    - 관련 외부 링크 (GitHub 저장소, 문서 등) 나열.
5.  **Disclaimer (주의사항)**:
    - 맨 마지막에 GitHub Alert 블록 (`> [!CAUTION]` 또는 `> [!WARNING]`) 사용.
    - 외부 스크립트나 AUR 패키지 설치 전 PKGBUILD 등을 확인할 것을 권고하는 문구 포함.

## 3. 태그 (Tags)
- **최소한으로, 구체적인 것만**: 기술명 위주로 작성 (예: "Linux", "Arch", "KakaoTalk", "Wine").
- **일반적인 태그 금지**: "Tips", "Guide", "Tutorial" 같은 포괄적이고 모호한 태그 사용 금지.
- **언어**: 영어/한글 상관없으나 기존 태그와 일관성 유지.

## 4. 예시 레이아웃

```markdown
---
title: "제목 입력"
summary: "짧은 요약."
date: 2026-01-27
tags: ["Linux", "ToolName"]
---

## TL;DR

### 빠른 설치
\`\`\`bash
command here
\`\`\`

## 본문

여기에 설명 작성.
...함. ...임.

### 소제목
내용...

## References

- [링크 이름](URL)

> [!CAUTION]
> 스크립트 실행 전 반드시 내용을 확인하세요.
```
